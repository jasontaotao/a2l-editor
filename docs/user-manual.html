a2l-editor 实操手册 v0.20

面向标定工程师的实战教程。从打开文件到合并变更，按场景一步一步来。

📖 场景一：打开与浏览 A2L 文件

你有一份来自 ECU 开发工具的 .a2l 文件，想查看里面的信号定义。

1
启动 GUI

打开命令行，执行：

dotnet run --project src/A2lEditor.App

或者双击 publish/a2l-editor.exe（如果已打包）。

2
打开文件

菜单 File → Open...，选择你的 .a2l 文件。
或者直接把文件从资源管理器拖到窗口上（窗口会显示蓝色光晕提示）。

3
浏览信号

左侧树形面板列出所有 MODULE，展开后可以看到 MEASUREMENT 和 CHARACTERISTIC 列表。

点击任意信号名，编辑器自动跳转到对应行并高亮半秒。

4
搜索内容

按 Ctrl+F 弹出搜索面板。输入关键字（如信号名、地址），支持上下翻遍历匹配项。点击面板上的替换按钮可切换替换模式。

💡 快捷键备忘
Ctrl+Z撤销
Ctrl+Y重做
Ctrl+F查找/替换
Ctrl++ / -放大/缩小
Ctrl+0重置缩放
F5验证文件

📖 场景二：验证文件是否正确

你编辑完 A2L 后，想检查有没有语法错误。

方法 A：在 GUI 中

按 F5 或菜单 Tools → Validate。
底部面板会显示所有错误和警告。双击任意错误行，编辑器跳到对应位置。

方法 B：在命令行中

# 验证文件
dotnet src/A2lEditor.Cli/bin/Release/net8.0/a2l-editor.dll validate samples/BmsModel.a2l

# 退出码：
#   0 = 通过，1 = 有错误/警告，2 = 文件打不开

# 如果你在项目目录下，也可以：
dotnet run --project src/A2lEditor.Cli -- validate samples/BmsModel.a2l
✅ 验证器检查项： 解析语法错误 · 重复信号名称 · 重复 VERSION 块 · 非 MSB_LAST 字节序警告。

📖 场景三：从 Excel 生成 A2L 骨架

你的标定团队用 Excel 管理信号列表，现在要生成一份 A2L 文件用于 ECU 集成。

1
准备 Excel 文件

按以下列布局整理数据（首行为表头，自动跳过）：

列内容示例
A信号名称V
B描述Battery voltage
C类型（可空，默认 MEASUREMENT）MEASUREMENT
D数据类型UBYTE
E转换方法BMS_CM_uint8
F下限0
G上限255
HECU 地址（十六进制）0x1000
I分辨率0.1
J精度1.0
2
生成 A2L

方法 A：命令行

dotnet run --project src/A2lEditor.Cli -- skeleton generate signals.xlsx -o skeleton.a2l

# 也可以指定模块名和 Sheet
dotnet run --project src/A2lEditor.Cli -- skeleton generate signals.xlsx \
    --module BMS_Module --sheet Sheet1 -o bms.a2l

方法 B：GUI（推荐初学者）

菜单 Tools → Import from Excel...

弹出对话框中：

  1. 点 Browse... 选择 .xlsx 文件
  2. 点 Preview 预览解析出的信号数量
  3. 确认 Module 名称和注释（可选修改）
  4. 点 Save As... 保存生成的 .a2l 文件
💡 自动生成的内容： MEASUREMENT / CHARACTERISTIC 声明 · COMPU_METHOD 存根（线性转换、直通系数） · 按数据类型匹配的 RECORD_LAYOUT。

📖 场景四：从 A2L 导出回 Excel

你手上有一份完整的 A2L 文件，需要提取信号列表给测试团队做分析。

方法 A：命令行

dotnet run --project src/A2lEditor.Cli -- skeleton export BmsModel.a2l -o BmsModel_signals.xlsx

方法 B：GUI

菜单 Tools → Export to Excel...，选保存路径即可。
前提：当前已打开一个有效的 .a2l 文件。

导出后，这个 xlsx 可以直接用 Import from Excel... 再转回 A2L：

dotnet run --project src/A2lEditor.Cli -- skeleton generate BmsModel_signals.xlsx -o roundtrip.a2l

📖 场景五：对比两个 A2L 文件的差异

你收到了两份 A2L 版本（比如 baseline 是旧版，compared 是同事修改后的新版），想找出具体改动。

方法 A：命令行

dotnet run --project src/A2lEditor.Cli -- diff compare baseline.a2l modified.a2l

# 只看摘要
dotnet run --project src/A2lEditor.Cli -- diff compare baseline.a2l modified.a2l --brief

方法 B：GUI

  1. 菜单 Tools → Diff Files...
  2. 在 Baseline 行点 Browse... 选旧文件
  3. 在 Compared 行点 Browse... 选新文件
  4. 点 Compare — 结果输出到下方文本框
  5. 差异按块类型分组：Added（新增）、Removed（删除）、Modified（修改的字段及新旧值）

📖 场景六：合并两份 A2L

你要把同事的修改合并到自己的 baseline 上，或者想选择性接受某些改动。

快速合并（全盘接受修改）

dotnet run --project src/A2lEditor.Cli -- merge apply baseline.a2l modified.a2l -o merged.a2l

# 先预览（不生成文件）
dotnet run --project src/A2lEditor.Cli -- merge apply baseline.a2l modified.a2l --dry-run

交互式合并（逐个选择）

  1. GUI 菜单 Tools → Diff Files...
  2. 选好两个文件后点 Compare
  3. 点 Save Merged...
  4. 弹出 Merge Conflict Review 审查窗口：
    🟢 Added 新增 / 🔵 Modified 修改 / 🔴 Removed 删除
    每项前面有复选框，取消勾选 = 跳过这个变更（保留 baseline 版本）
  5. 点 Accept Selected → 选保存路径 → 生成合并后的文件
💡 Removed 默认不勾选（删除操作不自动应用，需要手动确认）。

📖 场景七：从 MAP/ELF 更新 ECU 地址

固件编译后生成了新的 MAP/ELF 文件，你需要把里面的符号地址更新到 A2L 中。

# 先看看 MAP 文件里有哪些符号
dotnet run --project src/A2lEditor.Cli -- map dump-symbols firmware.map

# 预览地址匹配情况（不修改 A2L）
dotnet run --project src/A2lEditor.Cli -- map validate firmware.map target.a2l

# 执行更新（默认 dry-run，加 --backup 备份原文件）
dotnet run --project src/A2lEditor.Cli -- map update firmware.map target.a2l --output updated.a2l --backup

GUI 方式：菜单 Tools → Apply MAP...，选择 MAP/ELF 文件后会弹出覆盖预览（匹配数/缺失数/多余数），确认后写入。

📖 场景八：备份/交换 A2L 为 JSON

你想把 A2L 文件内容导出为 JSON，方便程序读取、版本对比或 CI 集成。

JSON 序列化是代码库 API（目前无 CLI 命令）：

// C# 代码
using A2lEditor.Core.Serialization;

var serializer = new A2lDocumentSerializer();
var doc = A2lDocument.LoadFromFile("BmsModel.a2l");

// 写 JSON
string json = serializer.SerializeToJson(doc);
File.WriteAllText("BmsModel.json", json);

// 从 JSON 读回
var docBack = serializer.DeserializeFromJson(File.ReadAllText("BmsModel.json"));

// 也支持 XML
string xml = serializer.SerializeToXml(doc);

JSON 输出特点：
枚举输出为名称（"UBYTE"、"V1_31"）而非数字，缩进格式化，非 ASCII 字符保持原样。

🔧 常用问题

Q: 打开文件时报解析错误？
检查文件编码是否为 UTF-8（a2l-editor 自动检测 UTF-8/16/32）。如果是其他编码，可以先用记事本另存为 UTF-8。
Q: Excel 导入时报找不到 Sheet？
确认 Sheet 名称大小写完全匹配。可用 --sheet 指定；如果不指定，默认取第一个 Sheet。
Q: 合并后某些修改没生效？
在交互式合并中，只有勾选了的条目才会应用。没勾选的修改保持 baseline 的旧值。取消勾选 Added 条目的结果是该块不会出现在合并结果中。
Q: 构建报错？
确认已安装 .NET 8 SDK（dotnet --version 检查）。首次构建可能需要 dotnet restore。
